## 1. 项目简介

**AgentOS** 是一个多智能体协作平台——用户可以创建多个专业化 AI Agent（研究员、程序员、写作者、整理师等），它们共享工具和记忆，协作完成复杂任务。

- **移动端 (React Native / Expo)**：Chat 指挥中心，对话式派发任务，实时追踪 Agent 执行进度
- **Web 管理后台 (React)**：Agent 工厂、工具注册、任务模板、监控日志
- **AI 核心服务 (Python / FastAPI)**：Agent 运行时、工具系统、记忆系统、多 Agent 编排
- **API 网关 (Node.js / NestJS)**：用户认证、流式推送、权限控制

## 4. 技术栈

| 层         | 技术                                 | 选择理由                                     |
| ---------- | ------------------------------------ | -------------------------------------------- |
| 移动端     | React Native (Expo)                  | JS 生态，AI SDK 兼容最好                     |
| Web 后台   | React + Vite + shadcn/ui             | 快速开发，组件丰富                           |
| API 网关   | NestJS (Node.js)                     | TypeScript 全栈，依赖注入优雅                |
| AI 核心    | Python FastAPI + LangChain           | AI 生态最成熟                                |
| 数据库     | PostgreSQL + pgvector                | 关系型+向量检索二合一                        |
| 缓存/队列  | Redis + BullMQ                       | 会话、限流、异步任务                         |
| 对象存储   | MinIO (S3 兼容)                      | 本地开发无需云服务                           |
| LLM 供应商 | OpenAI + Anthropic + Ollama          | 混合策略：小任务本地模型，复杂任务闭源大模型 |
| 部署       | Docker Compose (本地) + Railway (云) | 简单可迁移                                   |
| Monorepo   | Turborepo + pnpm workspace           | 统一管理前后端                               |

## 5. 技术决策记录

| 决策          | 选择                  | 理由                                            | 代价                                        |
| ------------- | --------------------- | ----------------------------------------------- | ------------------------------------------- |
| 移动端框架    | React Native (Expo)   | JS 生态与 AI SDK 兼容最好，全栈语言统一         | 复杂动画/原生性能不如 Flutter               |
| AI 语言       | Python                | AI/ML 生态统治地位，LangChain/LlamaIndex 最成熟 | 与前端语言不同，增加 context-switching 成本 |
| 混合后端      | Python+Node.js 双服务 | 各取所长：AI 走 Python，API 网关走 Node.js      | 运维复杂度增加，需维护两套服务              |
| 模型策略      | 混合(本地+云端)       | 降低成本，日常小模型足够，复杂任务再用大模型    | 模型路由逻辑需要精心设计                    |
| 向量数据库    | pgvector              | 无需额外部署服务，与业务库统一                  | 大规模场景性能不如 Milvus/Pinecone          |
| Monorepo 工具 | Turborepo             | 成熟稳定，构建缓存好，Vercel 维护               | 不支持非 JS 项目（Python 服务需单独处理）   |

## 6. MVP 范围（第一版交付）

**目标**：6-8 周完成，可演示、可放简历

```
用户 → 移动端 Chat → 创建任务
                    → API Gateway 转发
                    → AI Core 调度单个 Agent + 工具
                    → 结果推回移动端
```

**MVP 包含**：

- 1 种 Agent（通用助理，能搜索+总结+写代码）
- 3 个工具（Web 搜索、代码执行、文件读写）
- 移动端 Chat（核心交互界面）
- Web 后台精简版（Agent 配置 + 任务历史）
- 对话级短期记忆

**MVP 不包含**（Sprint 3+ 再做）：

- 多 Agent 协作编排
- 长期记忆 / 知识库
- 语音录入
- Widget
