# AgentOS - Claude 工作指南

## 项目概述

AgentOS 是一个多智能体协作平台：多个专业化 AI Agent 共享工具与记忆，协作完成复杂任务。技术栈：

- **移动端**: React Native (Expo)
- **Web 后台**: React + Vite + shadcn/ui
- **API 网关**: Node.js / NestJS
- **AI 核心**: Python / FastAPI + LangChain
- **数据**: PostgreSQL + pgvector, Redis, MinIO
- **Monorepo**: Turborepo + pnpm workspace

## 工作方式

### 核心原则

本项目采用**文档驱动迭代开发**——每个阶段用一份独立的 `.md` 文件追踪，形成记忆链。每次新对话，Claude 应该：

1. **先读取 `00-项目总览.md`** 了解项目全景
2. **找到当前进行中的阶段文档**（状态为 🟢 的文档）
3. **读取当前阶段文档** 获取上下文
4. **继续推进** 或 **协助修改计划**

### 文档规范

每个阶段的 `.md` 文档统一使用以下结构（参见 `00-项目总览.md` 模板）：

```markdown
# [阶段名称]

| 属性     | 值          |
| -------- | ----------- |
| 创建日期 | YYYY-MM-DD  |
| 最后更新 | YYYY-MM-DD  |
| 状态     | 🟢/🟡/✅/❌ |
| 上一阶段 | [链接]()    |
| 下一阶段 | [链接]()    |

## 1. 本阶段目标

## 2. 技术决策记录

## 3. 项目进度 (已完成/进行中/未开始)

## 4. 主要内容与代码结构

## 5. 当前阻塞 & 待解决问题

## 6. 不足与修改计划

## 7. 关键命令

## 8. 已产出物

## 9. 下一阶段计划
```

### 文档链

```
00-项目总览 → 01-项目初始化 → 02-AI核心-单Agent运行时
    → 03-记忆与知识系统 → 04-多Agent编排
    → 05-API网关 → 06-Web管理后台 → 07-移动端App
    → 08-集成联调与打磨
```

### 状态符号

- 🟢 进行中（当前阶段）
- 🟡 暂停（等待某条件）
- ✅ 已完成
- ⬜ 未开始

### 迭代规则

1. **当前阶段完成后**，将状态改为 ✅，`最后更新` 改为当日日期
2. **进入下一阶段时**，创建新 `.md` 文档，状态标 🟢
3. **更新 `00-项目总览.md`** 中文档链表格的状态列
4. **每次修改 `.md` 文件后** 需先获得用户确认再继续
5. **新对话开始时** 自动检查所有文档状态，给出"应该继续推进哪个阶段"的建议

### 文档位置

所有 `.md` 文件存放在项目根目录（`D:\项目\AgentOS\`）。
