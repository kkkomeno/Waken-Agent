## 1. 本阶段目标

搭建 AgentOS 完整的开发基础设施，确保四个子项目能在一条命令下并行启动，为后续 Sprint 提供坚实基底。

## 2. 技术决策记录

| 决策          | 选择                 | 理由                                                      | 代价                                     |
| ------------- | -------------------- | --------------------------------------------------------- | ---------------------------------------- |
| 包管理器      | pnpm                 | 磁盘效率高、strict mode 避免幽灵依赖、原生 workspace 支持 | 团队成员需额外安装 pnpm                  |
| Monorepo 工具 | Turborepo            | 构建缓存优秀、与 Vercel 生态集成好、配置简单              | Python 服务无法被 turbo 管理，需独立脚本 |
| Node 版本     | 22 LTS               | 最新长期支持版，ES2024 特性                               | Expo 可能有兼容性延迟                    |
| Python 版本   | 3.12                 | 最新稳定版，FastAPI 完全支持                              | 某些 AI 库可能滞后 1-2 月                |
| 前端 UI 库    | shadcn/ui            | 复制即拥有、Tailwind 生态、可定制性强                     | 组件数量不如 Ant Design                  |
| 代码规范      | ESLint flat config   | ESLint 9.x 新标准                                         | 旧 IDE 插件可能不兼容                    |
| commit 规范   | Conventional Commits | 社区标准、changelog 自动生成                              | 提交时有学习成本                         |

### 4.2 核心配置文件设计

#### pnpm-workspace.yaml

```yaml
packages:
  - 'apps/*'
  - 'packages/*'
```

#### turbo.json

```json
{
  "$schema": "https://turbo.build/schema.json",
  "globalDependencies": ["**/.env.*local"],
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**", ".next/**", ".expo/**"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "lint": {
      "dependsOn": ["^build"]
    },
    "typecheck": {
      "dependsOn": ["^build"]
    },
    "test": {
      "dependsOn": ["build"],
      "inputs": ["src/**/*.ts", "test/**/*.ts"]
    }
  }
}
```

#### docker-compose.yml

```yaml
services:
  postgres:
    image: pgvector/pgvector:pg16 # PostgreSQL 16 + pgvector
    environment:
      POSTGRES_USER: agentos
      POSTGRES_PASSWORD: agentos_dev
      POSTGRES_DB: agentos
    ports:
      - '5432:5432'
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./docker/postgres/init.sql:/docker-entrypoint-initdb.d/init.sql

  redis:
    image: redis:7-alpine
    ports:
      - '6379:6379'

  minio:
    image: minio/minio
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    ports:
      - '9000:9000'
      - '9001:9001'
    volumes:
      - minio_data:/data

volumes:
  postgres_data:
  minio_data:
```

### 4.3 共享类型包核心类型 (前期设计)

```typescript
// packages/shared/src/types/agent.ts

export type AgentStatus = 'idle' | 'running' | 'waiting' | 'done' | 'error';

export type AgentRole = 'researcher' | 'coder' | 'writer' | 'organizer' | 'general';

export interface AgentConfig {
  id: string;
  name: string;
  role: AgentRole;
  systemPrompt: string;
  model: ModelConfig;
  tools: string[]; // tool IDs
  permissions: AgentPermission[];
  maxSteps: number; // max ReAct loop steps
  timeout: number; // timeout in seconds
}

export interface ModelConfig {
  provider: 'openai' | 'anthropic' | 'ollama';
  modelName: string;
  temperature: number;
  maxTokens: number;
}

// packages/shared/src/types/task.ts

export type TaskStatus = 'pending' | 'decomposing' | 'running' | 'waiting_input' | 'done' | 'error';
export type CollaborationMode = 'pipeline' | 'parallel' | 'debate';

export interface Task {
  id: string;
  userId: string;
  title: string;
  description: string;
  status: TaskStatus;
  collaborationMode: CollaborationMode;
  subTasks: SubTask[];
  agents: string[]; // assigned agent IDs
  createdAt: string;
  completedAt?: string;
}

export interface SubTask {
  id: string;
  parentTaskId: string;
  assignedAgent: string;
  status: TaskStatus;
  input: string;
  output?: string;
  steps: AgentStep[];
}

export interface AgentStep {
  id: string;
  type: 'think' | 'act' | 'observe';
  content: string;
  toolCall?: ToolCall;
  timestamp: string;
}
```

### 4.4 开发工作流

```
# 首次启动
git clone <repo>
cd AgentOS
pnpm install
docker compose -f docker/docker-compose.yml up -d
cp .env.example .env
pnpm dev         # 启动所有服务

# 日常开发
docker compose -f docker/docker-compose.yml up -d   # 启动基础设施
pnpm dev --filter=@agentos/api                       # 只启动某一个
pnpm lint                                           # 全量 lint
pnpm typecheck                                      # 全量类型检查
pnpm test                                           # 全量测试
```

## 5. 当前阻塞 & 待解决问题

- [ ] **Git 仓库初始化** — 需要确认在 GitHub 还是 GitLab 创建仓库，仓库名确认
- [ ] **Python AI 服务的 turbo 集成** — Turborepo 原生不支持 Python，需要自定义脚本桥接。方案：在 `apps/ai-core/package.json` 做一个薄壳包装，`pnpm dev` → `uvicorn main:app`
- [ ] **Expo 与 Turborepo 兼容性** — Expo 有自己的 Metro bundler，与 turbo 的 `dev` task 可能有冲突，需要验证
- [ ] **Windows 开发环境** — Docker Desktop 在 Windows 上的文件挂载性能，MinIO 的 Windows 兼容性需要验证

## 7. 关键命令

```bash
# === 环境初始化 ===
pnpm install                              # 安装所有依赖
docker compose -f docker/docker-compose.yml up -d  # 启动数据库/缓存/存储
cp .env.example .env                      # 配置环境变量

# === 开发 ===
pnpm dev                                  # 一键启动所有服务 (Turbo)
pnpm dev --filter=@agentos/mobile         # 只启动移动端
pnpm dev --filter=@agentos/web            # 只启动 Web 后台
pnpm dev --filter=@agentos/api            # 只启动 API 网关
cd apps/ai-core && uvicorn app.main:app --reload  # 启动 AI 核心 (Python)

# === 代码质量 ===
pnpm lint                                 # 全量 ESLint
pnpm lint --fix                           # 自动修复
pnpm typecheck                            # 全量 TypeScript 类型检查
pnpm format                               # Prettier 格式化

# === 测试 ===
pnpm test                                 # 全量测试
pnpm test --filter=@agentos/api           # 单包测试

# === Docker ===
docker compose -f docker/docker-compose.yml down    # 停止所有服务
docker compose -f docker/docker-compose.yml logs -f # 查看日志
docker compose -f docker/docker-compose.yml down -v # 停止并清除数据
```
